底是怎么回事，使用起来流程是什么，其实aidl就是一个代码生成的模板，因为binder的类基本都差不多，有个stub和proxy这样的内部类和一些switch里
选择要调用的方法，为了方便上层app人员开发，所以定制了aidl这样的模板，按照这个模板开发，然后编译下会自动生binder的java文件，然后可以使用
进程间通信了。还有就是AMS和应用间的通信，其实是应用主动把自己的binder代理对象的句柄ApplicationThread传递给AMS，然后AMS就可以使用这个代理
binder对象调用应用进程的一些方法了，如调用Activity的生命周期的一些方法什么的，还有传递手势，到了应用进程这边再通过handler转化消息添加到
主线程的消息队列中，再传递给当前的运行中的Activity，由window来处理手势和点击事件的分发。
